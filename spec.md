# Technical Specification: FinOps AI System

## 1. System Architecture
The system follows a modular, agent-driven architecture:

```text
[ User Interface (Chainlit) ]
         |
         v
[ LangGraph Orchestrator (Manager) ]
    |------------|-------------|
    v            v             v
[ Vision ]   [ Graph ]     [ Math/Code ]
[ Engine ]   [ Engine ]    [ Engine ]
```

## 2. Core Technology Stack (100% OSS)
* **Language Model:** Llama-3 (8B) or Qwen-2.5 (Math variant) served via **Ollama** for local, private processing.
* **Visual Model:** **ColPali** (Vision Retrieval) + **Pixtral** (for reading the retrieved images).
* **Vector Database:** **Qdrant** (Stores ColPali page embeddings).
* **Graph Database:** **Neo4j Community Edition** (Stores the Temporal Knowledge Graph).
* **Frameworks:** **LangGraph** (Agent routing & state management), **Chainlit** (UI).

## 3. Data Schema: Temporal Knowledge Graph
Nodes and relationships must include time bounds to ensure accurate historical comparisons.

**Nodes:**
* `Company` (e.g., Name: Tata Motors, Ticker: TATAMOTORS)
* `Metric` (e.g., Name: EBITDA, Value: 14000, Unit: INR_Crores)
* `Segment` (e.g., Name: Commercial Vehicles)

**Temporal Relationships (Edges):**
* `(Company)-[REPORTED {quarter: "Q3", year: 2025, valid_from: "2024-10-01", valid_to: "2024-12-31"}]->(Metric)`
* `(Company)-[COMPETES_WITH {status: "active"}]->(Company)`
* `(Company)-[OPERATES_IN]->(Segment)`

## 4. Multi-Agent Definitions (LangGraph Nodes)
* **`Router Node`:** Uses an LLM to classify the user query into `visual_search`, `graph_search`, or `hybrid_search`.
* **`Vision Retriever Node`:** Queries Qdrant with the user's text. Qdrant returns the most relevant PDF page image. Passes the image to the Pixtral VLM to extract the answer.
* **`Graph Retriever Node`:** Translates the user query into a Cypher query. Executes against Neo4j to get exact historical numbers.
* **`Code/Math Node`:** If the user asks for a calculation ("What is the margin difference?"), this node generates a secure Python script, runs it via Python REPL, and returns the exact number.
* **`Synthesizer Node`:** Takes the outputs from the retrievers and the math node, and writes a professional financial summary.

## 5. UI/UX Specification (Chainlit Premium Theme)
To achieve an "Enterprise Grade" feel, we will override the default Chainlit theme using `.chainlit/config.toml` and custom CSS (`public/custom.css`).

**Color Palette:**
* **Background (App):** Deep Slate (`#0B132B`) - Reduces eye strain, looks professional.
* **Background (Chat Canvas):** Dark Navy (`#1C2541`) - Subtle contrast for the main workspace.
* **Primary Accent:** Muted Teal (`#3A506B`) or Gold (`#D4AF37`) - Used for user messages and primary buttons.
* **Text Color:** Off-white/Silver (`#E0E0E0`) - High readability without the harshness of pure white.

**Custom CSS Guidelines:**
* **Typography:** Use a clean, sans-serif font like `Inter` or `Roboto`.
* **Avatars:** Use custom, professional icons for the User and the AI (e.g., a stylized pie chart or sparkline for the AI).
* **Step Visibility:** Chainlit allows showing the "thought process." Style the LangGraph steps (e.g., "Querying Database...", "Calculating Ratios...") as collapsible, greyed-out accordions so the user *knows* the AI is doing complex work, building trust.
* **Markdown Support:** Ensure tables generated by the AI in the chat UI have clear borders, subtle hover effects on rows, and right-aligned numerical columns.
